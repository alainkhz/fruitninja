# 關卡系統設計

## 1. 目標

目前遊戲是無限生存節奏，玩久了會比較像純刷分。  
加入關卡系統後，核心目標改成：

- 玩家達到指定分數後進入下一關
- 每一關有明確難度提升
- 玩家會感受到節奏變化，而不是單純水果一直出
- 關卡切換要有短暫的成就感與提示

這套設計先以 MVP 為主，不先做複雜地圖或 Boss。

## 2. 核心規則

### 2.1 升關條件

- 每一關都有一個 `目標分數`
- 當玩家分數 `>= 目標分數` 時，立即進入下一關
- 升關時不重置：
  - 分數
  - 生命
  - Combo
  - 技能條
- 升關時會重置或更新：
  - 關卡目標分數
  - 生成節奏
  - 炸彈機率
  - 特殊水果權重

### 2.2 關卡顯示

HUD 新增：

- `關卡：1`
- `下一關：150 / 200`

意思是：

- 目前在第 1 關
- 還差多少分進入下一關

### 2.3 升關瞬間效果

當升關發生時：

- 畫面中央顯示 `第 2 關`
- 畫面短暫閃光 / tint
- 播放一個升關音效
- 所有場上水果清空或短暫凍結 `0.5 秒`

MVP 建議先用：

- 顯示升關字樣
- 播升關音效
- 場上水果清空

這樣最簡單也最乾淨。

## 3. 關卡節奏設計

### 3.1 難度提升方向

每升一關，至少調整這幾項：

- 水果生成間隔變短
- 水果飛行速度變快
- 炸彈出現率微幅提高
- 特殊水果出現率微調

不要一次拉太兇，不然體感會變成突然爆量。

### 3.2 建議公式

先用簡單線性方案：

- `關卡 = 1`
- `目標分數 = 200`
- 每升一關：
  - 下一關目標 `+150`
  - 生成間隔 `-0.05 秒`
  - 速度倍率 `+0.06`
  - 炸彈倍率 `+0.05`

並加上安全上限：

- 最短生成間隔不低於 `0.45 秒`
- 速度倍率不高於 `1.75`
- 炸彈倍率不高於 `1.8`

## 4. 範例關卡表

### 第 1 關

- 目標分數：`200`
- 玩法定位：暖身
- 特性：
  - 水果較慢
  - 炸彈較少
  - 普通水果為主

### 第 2 關

- 目標分數：`350`
- 玩法定位：開始要求連續切
- 特性：
  - 生成變快
  - 金色 / 冰凍水果出現率微升

### 第 3 關

- 目標分數：`500`
- 玩法定位：開始進入技能節奏
- 特性：
  - 火焰水果與電擊水果更常出現
  - 更適合靠技能刷分

### 第 4 關

- 目標分數：`700`
- 玩法定位：壓力提升
- 特性：
  - 炸彈更多
  - 水果速度更快

### 第 5 關+

- 後續關卡繼續按公式推進
- 不必先手寫到 10 關，MVP 可以自動生成

## 5. 關卡資料結構建議

可以新增一個 `LevelState` 或直接放進 `GameState`：

```python
level: int = 1
next_level_score: int = 200
spawn_interval_multiplier: float = 1.0
velocity_multiplier: float = 1.0
bomb_chance_multiplier: float = 1.0
level_transition_until: float = 0.0
```

## 6. 建議流程

### 6.1 每幀更新流程

1. 先正常更新分數
2. 檢查是否達到 `next_level_score`
3. 如果達到：
   - `level += 1`
   - 更新下一關目標
   - 更新生成參數
   - 顯示升關提示

### 6.2 生成器如何接入

目前生成器已有：

- 生成間隔
- 速度倍率
- 炸彈倍率
- 特殊水果權重

關卡系統只要額外提供一份 `level profile` 給生成邏輯即可。

例如：

```python
{
    "velocity_multiplier": 1.18,
    "bomb_chance_multiplier": 1.10,
    "gold_weight": 1.00,
    "freeze_weight": 1.05,
    "fire_weight": 1.10,
    "shock_weight": 1.12,
}
```

## 7. 模式相容設計

### 標準模式

- 完整啟用關卡系統

### 禪模式

- 可以保留升關
- 但炸彈維持不出現

### 挑戰模式

- 關卡成長更快
- 每關目標分數略高
- 升關加成也更明顯

## 8. UI 設計

### HUD

左上角新增：

- `關卡：3`
- `下一關：420 / 500`

### 升關提示

畫面中央：

- `第 4 關`
- `速度提升`
- `炸彈增加`

MVP 可以只顯示：

- `第 4 關`

### 結算畫面

可新增：

- `最高關卡：6`

並寫進歷史紀錄。

## 9. 歷史紀錄擴充

目前歷史紀錄可以再補：

- 本局最高關卡
- 達成升關次數

未來可以讓歷史頁多一欄：

- `最高關卡`

## 10. MVP 實作建議

第一版先做：

1. `level`
2. `next_level_score`
3. 達標升關
4. 升關中央提示
5. 升關後增加速度與生成壓力

先不要做：

- Boss 關
- 關卡地圖背景切換
- 關卡專屬水果
- 關卡專屬 UI 主題

## 11. 推薦參數

MVP 建議起始值：

```text
第 1 關目標分數：200
之後每關 +150
生成間隔每關 -0.05 秒
速度倍率每關 +0.06
炸彈倍率每關 +0.05
```

若實測太快：

- 目標分數改成每關 `+180`
- 或生成間隔改成每關只減 `0.03`

若實測太慢：

- 第 1 關改成 `150`
- 或第 2 關開始就加入更多特殊水果

## 12. 後續擴充方向

之後可以再升級成：

- 關卡主題制
  - 冰霜關
  - 火焰關
  - 電擊關
- 每 5 關一個精英關
- 每 10 關一個 Boss 關
- 關卡通關獎勵
  - 星星
  - 解鎖新技能
  - 解鎖新水果造型

## 13. 總結

這套關卡系統適合目前專案，因為：

- 不需要重寫整個遊戲
- 可以直接接到現有分數系統
- 可以直接接到現有水果生成邏輯
- 能明顯提升「有在推進」的遊玩感

一句話版本：

> 玩家靠達成分數門檻升關，每一關生成更快、速度更高、壓力更大，形成清楚的遊戲進程。
