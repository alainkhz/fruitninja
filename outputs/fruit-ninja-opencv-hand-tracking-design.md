# OpenCV Fruit Ninja with Camera Hand Tracking

## 1. 目標

用 Python + OpenCV 製作一個類似 Fruit Ninja 的小遊戲，但不使用滑鼠或鍵盤切水果，改成透過鏡頭即時偵測玩家手指移動，將手指軌跡當作刀刃進行切割。

這一版只做設計，不進入實作。

## 2. 核心玩法

- 玩家站在鏡頭前，伸出食指控制刀刃
- 鏡頭持續追蹤手指尖座標
- 手指快速移動時，在畫面上形成切割軌跡
- 水果碰到切割軌跡時被切開並得分
- 炸彈碰到切割軌跡時遊戲失敗或扣大量分數
- 水果漏接太多次時遊戲結束

## 3. 系統拆分

整體可分成 6 個模組：

1. 攝影機輸入模組
2. 手部與手指追蹤模組
3. 遊戲邏輯模組
4. 碰撞判定模組
5. 畫面渲染模組
6. 狀態與參數管理模組

## 4. 模組設計

### 4.1 攝影機輸入模組

責任：

- 開啟 webcam
- 持續讀取影像 frame
- 視需要做水平翻轉，讓玩家操作方向更直覺
- 控制解析度與 FPS

輸入：

- 攝影機裝置編號
- 目標解析度

輸出：

- 即時影像 frame

注意事項：

- 建議先固定 640x480 或 1280x720
- 若效能不足，優先降低解析度而不是降低遊戲更新頻率

### 4.2 手部與手指追蹤模組

責任：

- 從 frame 中偵測手部
- 取得手部關鍵點
- 鎖定食指指尖座標
- 輸出可用於遊戲控制的平滑座標

建議技術：

- OpenCV 負責影像擷取與繪製
- 手部追蹤建議搭配 MediaPipe Hands

原因：

- 純 OpenCV 只靠膚色分割或輪廓法，容易受背景、光線、膚色影響
- MediaPipe 在手部關鍵點偵測上穩定很多，較適合遊戲控制

輸入：

- 攝影機 frame

輸出：

- `index_finger_tip = (x, y)`
- `hand_detected = true/false`
- `tracking_confidence`

後處理：

- 座標平滑化，避免刀刃抖動
- 可用移動平均或簡單低通濾波
- 可計算手指速度，決定是否啟用切割判定

### 4.3 遊戲邏輯模組

責任：

- 產生水果與炸彈
- 更新物件位置、速度、重力
- 管理得分、生命、遊戲狀態

主要狀態：

- `START_MENU`
- `PLAYING`
- `PAUSED`
- `GAME_OVER`

遊戲物件欄位建議：

- `type`: fruit 或 bomb
- `x, y`
- `vx, vy`
- `radius`
- `rotation`
- `spawn_time`
- `is_sliced`
- `is_alive`

物理規則：

- 物件從畫面下方向上拋出
- 受到重力影響後落下
- 離開畫面底部後判定為漏接

難度調整：

- 隨時間增加水果生成頻率
- 提高初速度範圍
- 混入更多炸彈

### 4.4 碰撞判定模組

責任：

- 將手指移動路徑視為切割線段
- 判斷線段是否與水果圓形碰撞

判定方式：

- 每一幀記錄食指前一點與目前點
- 形成一條 segment
- 用「點到線段距離」判斷是否切到水果圓形

建議條件：

- 只有當手指速度超過門檻時才算有效切割
- 避免玩家把手停在水果上就一直重複切中

結果：

- 水果被切中：加分、播放切開效果、產生碎片或果汁特效
- 炸彈被切中：扣分或直接結束

### 4.5 畫面渲染模組

責任：

- 疊加遊戲畫面到攝影機影像上
- 顯示水果、炸彈、刀刃軌跡、分數、生命值
- 顯示開始畫面與結束畫面

畫面元素：

- 即時鏡頭背景
- 水果與炸彈 sprite
- 手指軌跡光刀效果
- HUD 分數與剩餘生命

視覺建議：

- 刀刃軌跡保留最近 5 到 15 個點
- 用漸層線條呈現殘影
- 切中時可短暫閃白或噴汁

### 4.6 狀態與參數管理模組

責任：

- 集中管理遊戲參數
- 方便後續微調與測試

建議參數：

- 攝影機解析度
- 偵測信心門檻
- 手指速度門檻
- 水果生成間隔
- 重力大小
- 玩家生命數
- 炸彈出現機率

建議做法：

- 先放在 `config.py` 或 JSON 檔
- 後期再考慮做設定介面

## 5. 控制流程

### 5.1 每幀主流程

1. 讀取攝影機 frame
2. 做手部偵測與食指定位
3. 更新手指軌跡與速度
4. 更新所有遊戲物件位置
5. 檢查切割碰撞
6. 更新分數、生命、特效
7. 繪製整個畫面
8. 顯示到視窗

### 5.2 手指轉刀刃邏輯

1. 偵測到食指尖座標
2. 將鏡頭座標映射到遊戲畫面座標
3. 與前一幀座標形成線段
4. 若速度大於閾值，該線段為有效刀刃
5. 用此線段與所有水果做碰撞判定

## 6. 建議技術選型

### 最小可行組合

### 可選強化

## 8. MVP 範圍

第一版建議只完成以下能力：

- 開啟攝影機
- 偵測一隻手的食指尖
- 顯示手指軌跡
- 畫面中有水果往上拋出再掉落
- 手指揮過水果可得分
- 漏掉水果會扣生命
- 碰到炸彈會結束

先不要做：

- 雙手支援
- 真正的水果切半動畫
- 複雜粒子效果
- 選單動畫
- 網路排行榜

## 9. 技術風險與對策

### 9.1 光線不穩造成追蹤失敗

風險：

- 手指座標跳動
- 偵測不到手

對策：

- 優先使用 MediaPipe 關鍵點
- 加入平滑化
- 在畫面上提示玩家移到光線較穩的位置

### 9.2 手指移動太慢或太快

風險：

- 太慢時誤判切割
- 太快時座標跳點，切不到水果

對策：

- 設定最小切割速度門檻
- 軌跡保留最近數幀，避免單點判定

### 9.3 效能不足

風險：

- 遊戲延遲高
- 操作不順

對策：

- 降低攝影機解析度
- 每幀只追蹤必要資訊
- 控制同時存在的物件數量

### 9.4 鏡像方向讓玩家混亂

風險：

- 玩家往左揮，畫面看起來往右

對策：

- 預設做水平翻轉
- 保留參數可切換

## 10. 測試規劃

### 功能測試

- 鏡頭是否能正常開啟
- 單手是否能穩定抓到食指
- 手指軌跡是否連續
- 水果是否能正確生成與掉落
- 切到水果是否正確加分
- 切到炸彈是否正確結束

### 體驗測試

- 一般室內光線下是否可玩
- 背景複雜時是否仍穩定
- 不同距離下手指是否仍可辨識
- 延遲是否低到足以遊玩

## 11. 實作順序建議

1. 先完成攝影機畫面讀取
2. 接入手部偵測並抓出食指尖
3. 畫出手指軌跡
4. 做單顆水果生成與碰撞
5. 加入分數與生命系統
6. 加入炸彈與 game over
7. 最後再補素材、音效、特效與難度曲線

## 12. 後續可擴充方向

- 支援雙手連擊
- 特殊水果加分機制
- Combo 系統
- 切割角度影響特效
- 背景音樂與音效
- 全螢幕模式
- 關卡模式或生存模式

## 13. 創意玩法提案

除了基本的切水果玩法，這個專案很適合加入一些偏體感遊戲風格的創意機制，讓「用手指揮動」這件事更有表現力。

### 13.1 特殊技能

#### 時間緩速

觸發方式：

- 連續切中多顆水果後累積能量
- 能量滿時做一個快速橫向大揮手啟動

效果：

- 遊戲內物件移動速度暫時下降
- 玩家更容易連續切中多顆水果

設計價值：

- 適合補償鏡頭操作比滑鼠更難精準的問題
- 可做出很有成就感的連斬時刻

#### 狂暴刀刃

觸發方式：

- 短時間內達成 combo

效果：

- 刀刃變粗
- 有效碰撞範圍稍微放大
- 分數倍率提升

設計價值：

- 回饋玩家穩定操作
- 讓畫面更有爽感

#### 防爆護盾

觸發方式：

- 切中特定技能水果後獲得一次性護盾

效果：

- 下一次切到炸彈時不會直接死亡
- 只會消耗護盾

設計價值：

- 降低新手挫折感
- 增加策略選擇

#### 旋風斬

觸發方式：

- 玩家在短時間內畫出接近圓形的手勢

效果：

- 以玩家當前手指位置為中心產生範圍切割
- 清除附近水果

設計價值：

- 很適合鏡頭手勢互動
- 能把「追蹤路徑」升級成更有辨識度的技能輸入

### 13.2 特殊水果設計

#### 冰凍水果

效果：

- 切中後全場水果短暫減速

視覺：

- 淺藍色外框
- 切開時有冰晶碎裂效果

#### 火焰水果

效果：

- 切中後短時間內每次得分額外加成

視覺：

- 紅橘發光邊緣
- 軌跡會暫時變成火焰刀光

#### 閃電水果

效果：

- 切中一顆後會連鎖擊中附近水果

視覺：

- 黃白色閃電效果
- 水果之間出現電流連線

#### 分裂水果

效果：

- 切中後不是消失，而是分裂成 2 到 3 顆小水果
- 小水果分數較低，但容易組成 combo

設計價值：

- 提高場面混亂感
- 讓高手更容易做出高連段

#### 金色水果

效果：

- 高分稀有目標
- 切中可直接獲得大量分數或補生命

設計價值：

- 製造短暫的高價值決策
- 玩家會主動調整揮刀路線去追它

#### 毒水果

效果：

- 切中後畫面短暫模糊、反轉、或刀刃縮小

設計價值：

- 提供不是炸彈但仍有風險的負面物件
- 增加玩法層次

### 13.3 額外切割效果

#### 依揮刀速度決定特效強度

- 揮得越快，切開特效越大
- 可搭配額外分數獎勵

這樣可以鼓勵玩家真的做出有力的動作，而不是慢慢移動手指。

#### 依切割角度觸發不同效果

- 橫切：一般切割
- 直切：額外加分
- 斜切：更高機率觸發 combo 加成

這類設計可以讓玩家不只是「碰到就好」，而是追求更精準的操作。

#### 果汁殘留區域

- 某些水果切開後在畫面留下一小塊特效區域
- 區域存在期間，經過的水果可獲得加分或減速

這會讓遊戲多一點場域控制感，不只是一刀一刀反應。

### 13.4 手勢型技能輸入

因為這個版本是鏡頭追蹤手指，所以很適合把「手勢」直接設計成技能觸發方式，而不是只靠按鍵。

可考慮的手勢：

- 快速水平揮動：啟動時間緩速
- 畫圓：旋風斬
- 往上挑一下：空中爆發斬
- 連續左右來回：進入短暫狂暴模式

實作上要注意：

- 手勢辨識不能太嚴格，不然很難成功
- 技能判定要與一般切水果動作區隔，避免誤觸

### 13.5 分數與平衡建議

如果加入特殊水果和技能，建議同步調整分數系統：

- 普通水果：基礎分數
- 連擊：倍率提升
- 特殊水果：提供狀態效果，不一定直接高分
- 金色水果：高分獎勵
- 負面水果：造成 debuff，不一定直接 game over

平衡原則：

- 技能要幫助玩家創造更爽的連擊，而不是直接自動清場太久
- 特殊水果出現頻率不能太高，不然普通水果會失去存在感
- 負面效果應該製造緊張感，但不能讓玩家覺得被亂搞

### 13.6 適合第一波加入的創意元素

如果之後要從 MVP 往上加，建議先加這 4 個，成本和效果最平衡：

1. 冰凍水果
2. 金色水果
3. 狂暴刀刃
4. 簡單 combo 倍率

這四項不需要太複雜的辨識模型，就能明顯提升遊戲特色。

## 14. 結論

這個專案最合理的方向不是只靠純 OpenCV 做手指偵測，而是讓 OpenCV 負責影像處理與遊戲畫面，手部關鍵點追蹤則交給 MediaPipe。這樣可以大幅降低手部辨識的不穩定性，讓開發重點集中在遊戲手感、切割判定與畫面表現。

若下一步要開始實作，建議先從「攝影機 + 食指追蹤 + 軌跡顯示」這個最小閉環開始。
